Начало работы

Ссылка на раздел

Добро пожаловать в документацию API EagTek. Это руководство поможет вам начать работу с нашим API и интегрировать его в ваши приложения.

Быстрый старт

Для начала работы вам понадобится API ключ, который можно получить в личном кабинете разработчика.

Базовый URL

https:

Пример запроса 

curl -X GET \
  https:
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
fetch('https://api.eagtek.com/v1/users', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Accept': 'application/json'
  }
})
.then(response => response.json())
.then(data => console.log(data));
import requests

url = "https://api.eagtek.com/v1/users"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Accept": "application/json"

response = requests.get(url, headers=headers)
print(response.json())

Аутентификация

Ссылка на раздел

Все запросы к API EagTek должны быть аутентифицированы. Мы используем Bearer Token аутентификацию через заголовок HTTP.

POST /auth/token

Получение токена доступа

Для получения токена доступа отправьте POST запрос с вашими учетными данными.

Параметры запроса

Параметр Тип Описание
email string Email пользователя *
password string Пароль пользователя *

Пример запроса

curl -X POST \
  https:
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"yourpassword"}'

Пример ответа

{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 3600,
  "refresh_token": "def50200e5b2a3d1c3a4a5b6c7d8e9f..."
}

Использование токена

После получения токена доступа, включите его в заголовок Authorization всех последующих запросов:

Authorization: Bearer YOUR_ACCESS_TOKEN

Безопасность

Никогда не раскрывайте ваш API ключ или токен доступа в клиентском коде или публичных репозиториях. Все запросы должны выполняться с серверной части вашего приложения.

Ошибки API

Ссылка на раздел

API EagTek использует стандартные HTTP коды статусов для индикации успешности или неудачи запроса.

Коды ошибок

Код Статус Описание
200 OK Запрос выполнен успешно
201 Created Ресурс успешно создан
400 Bad Request Неверные параметры запроса
401 Unauthorized Требуется аутентификация
403 Forbidden Нет прав доступа
404 Not Found Ресурс не найден
429 Too Many Requests Превышен лимит запросов
500 Internal Server Error Ошибка сервера

Формат ошибки

При возникновении ошибки API возвращает JSON объект с описанием проблемы:

{
  "error": {
    "code": "invalid_request",
    "message": "Неверные параметры запроса",
    "details": {
      "email": "Поле email обязательно для заполнения"
    }
  }
}

Лимиты запросов

Ссылка на раздел

Для обеспечения стабильной работы API мы применяем ограничения на количество запросов.

Текущие лимиты

100 запросов в минуту на один API ключ

1,000 запросов в час на один API ключ

Заголовки лимитов

Каждый ответ API содержит заголовки с информацией о текущих лимитах:

Заголовок Описание
X-RateLimit-Limit Общее количество разрешенных запросов в текущем периоде
X-RateLimit-Remaining Количество оставшихся запросов в текущем периоде
X-RateLimit-Reset Время (UNIX timestamp) сброса лимита

Превышение лимитов

При превышении лимитов API возвращает код 429. Для увеличения лимитов свяжитесь с нашей службой поддержки.

Пользователи

Ссылка на раздел

API для работы с пользователями позволяет создавать, читать, обновлять и удалять пользователей, а также управлять их правами доступа.

GET /users

Получить список пользователей

Возвращает список всех пользователей с пагинацией. Требует прав администратора.

Параметры запроса

Параметр Тип Описание
page integer Номер страницы (по умолчанию 1)
per_page integer Количество элементов на странице (по умолчанию 20, максимум 100)
status string Фильтр по статусу (active, suspended, pending)
role string Фильтр по роли (admin, user, manager)

Пример запроса

curl -X GET \
  https:
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

POST /users

Создать пользователя

Создает нового пользователя. Требует прав администратора.

Параметры запроса

Параметр Тип Описание
name string Полное имя пользователя *
email string Email адрес *
password string Пароль (мин. 8 символов) *
role string Роль (user, admin, manager)

Пример запроса

curl -X POST \
  https:
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Новый пользователь","email":"new@example.com","password":"securepassword123","role":"user"}'

Пример ответа

{
  "data": {
    "id": "usr_789",

    "email": "new@example.com",
    "avatar": null,
